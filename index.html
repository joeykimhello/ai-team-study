CSV·구글시트는 가장 쉬운 시작점

  • 구조가 단순합니다.
  • 표 형태라 이해하기 쉽습니다.

  • 내부 관리표 · 명단 · 일정표에 적합합니다.

AI 스터디 2주차07
GPT · Gemini 활용

AI에게 이렇게 물어보면 됩니다

API를 직접 검색하고 문서를 해석하기 어렵다면, 대화형으로 물어보며 진행하세요.

PROMPT 1 · API 찾기
나는 구글시트에 [원하는 데이터]를 자동으로 가져오고 싶어. 사용할 수 있는 공개 API가 있는지, 무료인지, 인증키가 필요한지, 구글시트 연동이 되는지 알려줘.
PROMPT 2 · 연결 방법
이 API를 Apps Script로 연결하고 싶어. 초보자도 이해할 수 있게 단계별로, 준비물·요청 URL·예시 코드·시트에 넣는 방법까지 알려줘.
PROMPT 3 · 항목 고르기
응답이 JSON으로 오는데, 이 중에서 구글시트에 넣기 좋은 항목만 골라줘. 컬럼명도 추천해줘.
PROMPT 4 · 표 구조
아래 API 응답 예시를 보고, 구글시트 표로 정리하려면 어떤 컬럼 구조가 좋을지 추천해줘.
AI 스터디 2주차08
좋은 질문의 구조

AI에게 질문할 때 포함하면 좋은 것

  • 가져오고 싶은 데이터
  • 사용 목적
  • 구글시트에 넣고 싶은 이유
  • 무료 API 선호 여부
  • 인증키 사용 가능 여부
  • 원하는 컬럼 구조
  • 초보자 기준 설명 요청
예시 질문
“나는 지역 행사 데이터를 구글시트에 정리하고 싶어. 행사명·날짜·장소·설명·링크를 컬럼으로 만들고 싶고, 무료 API를 우선으로 찾고 싶어. 사용할 수 있는 API와 연동 방법을 초보자 기준으로 알려줘.”
AI 스터디 2주차09
API 사용 시 주의사항

API를 쓸 때 꼭 확인할 것

  • 모든 API가 무료는 아닙니다.
  • 인증키가 필요한 경우가 많습니다.
  • 하루 요청 횟수 제한이 있을 수 있습니다.
  • 문서의 사용 조건을 꼭 확인합니다.
  • 개인정보·민감 데이터는 함부로 수집·저장 금지.
  • 응답이 복잡하면 필요한 항목만 골라냅니다.
  • 한 번에 너무 많이 가져오면 시트가 느려집니다.
🔑 API 키는 비밀번호처럼 다루기 — 공개 웹페이지나 공유 문서에 노출 금지
AI 스터디 2주차10
API vs 크롤링

API가 없으면 크롤링을 생각할 수 있습니다

🔌API
  • 공식적으로 제공되는 데이터 통로
  • 구조가 비교적 안정적
  • 자동화에 적합
  • 인증키·사용량 제한이 있을 수 있음
🕸️크롤링 / 자동화
  • 웹페이지에 보이는 정보를 자동 수집
  • API가 없을 때 대안으로 사용
  • 사이트 구조 변경에 취약
  • 약관·저작권·개인정보 이슈 확인 필요
“이번 주에는 크롤링이 아니라 API 사용을 실습합니다.”
